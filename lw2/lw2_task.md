# Лабораторная №2. JavaScript-приложение с прототипным наследованием/ООП.

Цель: изучить механизм прототипного наследования в JavaScript и методы рисования в браузере на canvas.

## Задание

Разработать программу на JavaScript с примернением ООП подхода, используя прототипное наследование.

1. Программа должна содержать классы Сircle, Rectangle, Triangle, унаследованные от класса Shape 

  - Свойства класса Shape
    * setFillColor(value), getFillColor()
    * setBorderColor(value), getBorderColor()

  - Методы класса Shape, которые определяются в унаследованных классах:
    * draw() - отрисовывает графический примитив на Canvas
    * calculateArea() - вычисляет площадь фигуры
    * calculatePerimeter() - вычисляет периметр фигуры

  - Свойства класса Circle
    * getRadius(), setRadius(value)
    * getX(), setX(value)
    * getY(), setY(value)

  - Свойства классов Rectangle и Triangle - аналогично

  - Все параметры фигур должны быть проинициализированы значениями по умолчанию при вызове конструктора. Например, при создании окружности она создается с центром в (0, 0), радиусом 0, цветом обводки и заливки #000000

2. Разработать графический интерфейс, позволяющий:

  - Выбрать фигуру для манипуляции с помощью выпадающего списка

  - Ввести с клавиатуры общие параметры фигуры:
    * Fill Color - цвет заливки фигуры
    * Border Color - цвет обводки фигуры

  - Ввести с клавиатуры параметры конкретной фигуры (конкретный интерфейс появляется после выбора фигуры): 
    * Radius/CenterX/CenterY для круга
    * X1/Y1/X2/Y2 для прямоугольника
    * X1/Y1/X2/Y2/X3/Y3 для треугольника

  - После каждого изменения вводимых параметров:
    * Фигура должна перерисоваться с учетом указанных параметров
    * На экране должны быть показаны площадь и периметр выбранной фигуры, отрисовать также на canvas

## Требования

  * Использование ООП подхода с прототипным наследованием
  * HTML/CSS/JavaScript файлы должны быть разделены. В HTML недопустимо использование CSS/JavaScript, в JavaScript недопустимо использование HTML/CSS. Каждый класс также должен находиться в отдельном файле
  * Отсутствие использования сторонних библиотек, "Vanilla JS"
  * Полное соотвествие кода правилам стиля в наборе правил eslint:recommended

## Требования по процессу

1. Сделать fork этого репозитория
2. Код должен находиться в отдельной ветке, отпочкованной от ветки `lw2`, в директории `/lw2/<фамилия>_<имя>/`
3. Много коммитов
4. После завершения работы сделать pull request вашей ветки в ветку `lw2` этого репозитория с заголовком `<фамилия> <имя> lw2`
